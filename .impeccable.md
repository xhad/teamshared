# TeamShared Design Context

## Users

Engineering leads, developers, and ops teammates who administer a shared agent memory system. They sign in briefly to review wiki topics, approve agent work, mint API keys, and triage the task queue — often from a laptop, sometimes from a phone between meetings.

## Brand Personality

**Clear · Grounded · Editorial**

A serious knowledge workspace for teams and agents — not a consumer app or AI demo. It should feel like a well-maintained internal tool: confident typography, calm surfaces, information density where it matters.

## Aesthetic Direction

- **Mode**: Light-first with a restrained dark variant via `prefers-color-scheme` (no glowing accents).
- **Palette**: Warm stone neutrals tinted toward olive; accent deep teal (`oklch` ~175° hue) for actions and active states.
- **Typography**: Instrument Sans (UI) + Source Serif 4 (page titles and editorial headings). Modular scale with `clamp()`.
- **Layout**: Fixed sidebar on desktop; slide-out drawer on mobile. Tables and lists over card grids. Generous whitespace, subtle borders — not drop-shadow rectangles everywhere.
- **Motion**: Short ease-out transitions for nav drawer and hover states only; respect `prefers-reduced-motion`.

## Anti-References

- Purple/indigo gradient dark dashboards
- Inter / system-ui-only stacks
- Hero metric card grids with identical rounded rectangles
- Glassmorphism topbars and neon status dots

## Design Principles

1. **Information first** — every screen leads with the task; decoration supports scanning.
2. **Responsive parity** — mobile gets the same capabilities via drawer nav and scrollable tables, not amputated views.
3. **Distinct but professional** — serif headlines and warm neutrals signal craft without flash.
4. **Accessible contrast** — body text meets WCAG AA on all surfaces; focus rings on every interactive control.
5. **Consistent tokens** — one CSS variable system shared across console, login, and public dashboard.
